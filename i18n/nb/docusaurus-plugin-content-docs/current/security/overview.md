Innebygde enheter er i økende grad tilkoblede og sikkerhetskritiske. Én eneste sårbarhet kan føre til fare for mennesker, tjenesteavbrudd, tap av IP eller manglende etterlevelse av regulatoriske krav. Ved å bygge inn sikkerhet fra start reduserer du risikoen gjennom hele livssyklusen og den totale eierkostnaden.

Viktige drivere:

- Safety: forhindre farlig oppførsel som følge av ondsinnede input eller feil.  
- Compliance: oppfylle nye reguleringer (for eksempel EU Cyber Resilience Act).  
- Resiliens: begrense skadeomfanget og muliggjøre sikker gjenoppretting.  
- Tillit: beskytte data, IP og integriteten i leverandørkjeden.  

Hovedpilarer:

- Threat modelling og sikker arkitektur  
- Root of Trust, secure boot og oppdateringsintegritet  
- Minnebeskyttelse og least privilege  
- Sikker kommunikasjon og nøkkelhåndtering  
- Sårbarhetshåndtering og incident response  

Neste steg: gå videre til CRA‑delen for å se hvilke forpliktelser utviklere har og hvordan de påvirker innebygde prosjekter. Se Security → CRA → References for offisielle lenker og standarder.

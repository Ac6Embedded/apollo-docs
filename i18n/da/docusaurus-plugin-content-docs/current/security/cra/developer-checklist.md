## Sådan bruger du denne checklist

Dette er en **levende checklist**, som bør ligge i dit repo (fx `doc/cra-checklist.md`). Den forbinder de daglige udviklingsopgaver med evidens for CRA bilag I, så hver release kan dokumentere konformitet uden panik i slutningen.[1]  

**Tommelfingerregel:** en boks er kun “færdig”, hvis den er knyttet til **evidens**:

- ticket/issue‑ID’er,  
- MR/PR‑links,  
- build‑artefakter og logfiler,  
- testrapporter,  
- SBOM + VEX (eller tilsvarende),  
- opdateret dokumentation, risikovurderinger og threat models.  

Markér punkter pr. release og link dem til sager, PR’er og dokumenter.

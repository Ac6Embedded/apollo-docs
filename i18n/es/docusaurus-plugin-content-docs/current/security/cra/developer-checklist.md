## Cómo usar esta lista

Esta es una **lista viva** pensada para vivir en tu repositorio (por ejemplo `doc/cra-checklist.md`). Conecta las tareas diarias de ingeniería con la evidencia del Anexo I del CRA, de modo que cada versión pueda demostrar conformidad sin prisas de última hora.[1]  

**Regla general:** una casilla solo está “terminada” si enlaza con **evidencia**:

- ID de ticket(s),  
- enlaces a MR/PR,  
- artefactos / registros de build,  
- informes de pruebas,  
- SBOM + VEX (o equivalente),  
- actualizaciones de documentación, análisis de riesgos y modelos de amenazas.  

Marca los elementos por versión y enlázalos a tickets, PRs y documentos.
